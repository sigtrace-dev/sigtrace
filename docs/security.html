Security Policy

Vulnerability disclosure guidelines and supported versions

Supported Versions

We actively maintain and patch security vulnerabilities for the following versions of SigTrace:

Version Supported
1.1.x 🟢 Supported (Active patches)
1.0.x 🟡 Critical fixes only
< 1.0 🔴 Unsupported

Reporting a Vulnerability

⚠️ Warning: Please do NOT open public GitHub issues for security vulnerabilities. Always report privately.

If you find a security issue in SigTrace, please report it privately through GitHub's secure vulnerability reporting tool:

Report Vulnerability Privately

Your report will be reviewed privately, and we will coordinate a fix and release with you before public disclosure.

Security Architecture

  • 100% Local: SigTrace runs its WebSocket server exclusively on localhost (127.0.0.1). Telemetry data never leaves your machine.
  • Development Only: Instrumentation is only injected during development. It is a complete no-op and is stripped out automatically in production builds.
